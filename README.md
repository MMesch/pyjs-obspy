# Obspy Pyjs Demo

A demo application showing how to use Obspy through Pyjs to fetch and plot seismic data in the browser.

## Local Development

### Prerequisites
- Python 3.11+
- Node.js (for some build tools)
- Git

### Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/jupyterlite-obspy.git
   cd jupyterlite-obspy
   ```

2. Install dependencies:
   ```bash
   pip install empack
   ```

3. Build the environment:
   ```bash
   ./build.sh
   ```

4. Start a local server:
   ```bash
   python -m http.server 8000
   ```

5. Open your browser to http://localhost:8000

## Deployment

The application is automatically built and deployed to GitHub Pages using GitHub Actions when you push to the `main` branch.

## How It Works

1. The application uses Pyjs (Python compiled to WebAssembly) to run Python code in the browser
2. It connects to the EPOSFR FDSN service to fetch seismic data from the CURIE station
3. The data is processed using Obspy and plotted using Matplotlib
4. The plot is converted to a base64-encoded PNG and displayed in the browser

## Files

- `index.html` - Main application interface
- `pyjs_runtime_browser.js` - Pyjs runtime (generated by build)
- `pyjs_runtime_browser.wasm` - WebAssembly binary (generated by build)
- `empack_env_meta.json` - Environment metadata (generated by build)
- `packages/` - Python packages (generated by build)
- `.github/workflows/deploy.yml` - GitHub Actions workflow for deployment

## Troubleshooting

### Local Development Issues

If you get errors about missing files:
- Make sure you've run `./build.sh`
- Check that all generated files are present

### GitHub Pages Issues

If the deployment fails:
- Check the GitHub Actions logs
- Make sure all build artifacts are being generated
- Verify that the GitHub Pages settings are configured correctly
