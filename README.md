# A static website using obspy through WASM

This project demonstrates how to use Obspy through Pyjs to fetch and plot seismic data from the French CURIE station directly in the browser using WebAssembly.

## Build Instructions

### Prerequisites
- [Micromamba](https://mamba.readthedocs.io/en/latest/installation/micromamba-installation.html) or Mamba/Conda
- [Empack](https://github.com/emscripten-forge/empack)

### Setup

1. **Create the conda environment:**
   ```bash
   micromamba create -f environment.yml --platform emscripten-wasm32 --prefix ./env
   ```

2. **Copy pyjs runtime files:**
   ```bash
   cp ./env/lib_js/pyjs/* .
   ```

3. **Pack the environment:**
   ```bash
   empack pack env --env-prefix ./env --outdir .
   ```

4. **Serve the application:**
   ```bash
   python -m http.server 8000
   ```

5. **Open in browser:**
   Navigate to `http://localhost:8000`

## Project Structure

After building, your directory should contain:
- `index.html`: Main HTML file
- `pyjs_runtime_browser.js`: Pyjs WebAssembly runtime
- `pyjs_runtime_browser.wasm`: Pyjs WebAssembly module
- `empack_env_meta.json`: Empack environment metadata (generated by empack)
- `packages/`: Directory containing package tarballs (generated by empack)
- `environment.yml`: Conda environment specification

## How It Works

1. Browser loads `index.html` which loads `pyjs_runtime_browser.js`
2. Pyjs runtime initializes WebAssembly
3. When you click "Fetch CURIE Station Data":
   - Pyjs loads the empack environment (Obspy + dependencies)
   - Executes Python code to fetch data from EPOSFR FDSN service
   - Creates matplotlib plot and encodes as base64 PNG
   - Displays the plot in the browser

## Environment Details

The `environment.yml` specifies:
- `pyjs`: The Python-JavaScript bridge
- `obspy`: Seismological data processing
- `matplotlib`: Plotting library
- `numpy`, `scipy`: Scientific computing dependencies
- `requests`: HTTP library for monkey patching urllib

## Technical Details

### HTTP Monkey Patching

Since WebAssembly Python environments don't support native HTTP requests, we monkey patch `urllib` to use the `requests` library. The Python code includes a `RequestsURLopener` class that:

1. Replaces `urllib.request.URLopener`
2. Uses `requests.get()` and `requests.post()` for HTTP operations
3. Returns file-like objects that mimic urllib's response interface
4. Includes fallback to JavaScript's `fetch` API via `pyjs.js.window.fetch`

This allows Obspy's FDSN client to work in the browser environment.

## Testing

You can test the obspy functionality locally before building for the browser:

```bash
# Install dependencies in a regular Python environment
mamba create -n test-obspy -c conda-forge obspy matplotlib numpy scipy
mamba activate test-obspy

# Run the test script
python test_obspy.py
```

This will verify that the obspy data fetching and plotting works correctly.

## Notes

- First load may be slow due to WASM initialization and environment loading
- Data is fetched from the last 3 hours from the French CURIE station via EPOSFR
- Only HHZ channel is plotted for simplicity
- The empack environment is self-contained and doesn't require network access after initial load
- For production deployment, consider hosting on GitHub Pages or other static hosting

## GitHub Pages Deployment

To deploy to GitHub Pages:

1. Build the project as described above
2. Commit the built files (or use GitHub Actions)
3. Enable GitHub Pages in your repository settings
4. Point to the `main` branch and `/` (root) directory

The demo will be available at `https://yourusername.github.io/pyjs-obspy/`
